Composite (компонувальник) – використовується для представлення об'єктів у вигляді деревоподібної структури. Клас Task є базовим класом для всіх завдань, а клас CompositeTask представляє композитне завдання, яке може містити інші завдання. Класи Task і CompositeTask дозволяють створювати ієрархію завдань, у якій кожне завдання може бути простою, або композитною, тобто. містити у собі інші завдання.

Iterator - використовується для послідовного доступу до елементів складових об'єктів без розкриття деталей їх реалізації. Клас TaskIterator дозволяє ітеруватися за дочірніми завданнями композитної задачі без розкриття деталей її реалізації.

Factory Method (фабричний метод) – використовується для створення об'єктів, приховуючи деталі їх створення від коду клієнта. Фабричний метод CreateTask у класі Task дозволяє створювати об'єкти класів Task та CompositeTask, приховуючи деталі їх створення від коду клієнта.
